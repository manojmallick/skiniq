# SkinIQ — Startup World Cup Hackathon Pitch & Demo Plan

This document outlines the optimal 3-minute demo flow to showcase all the "Winner-Tier" technical features of SkinIQ to the hackathon judges.

---

## ⏱️ 0:00 - The Hook (Upload Screen)
**Talking Track:** *"Welcome to SkinIQ. We bridge the gap between clinical dermatology and e-commerce. It starts with a simple selfie, but underneath, we've built a robust, privacy-first pipeline."*

**Actions:**
1. Show the **Upload Screen**.
2. Point out the **three trust badges** at the bottom: **"GDPR Compliant"**, **"HIPAA Ready"**, and **"No data stored"** — privacy-first architecture judges love to see.
3. Select French (or Dutch/Hindi) from the **Language Switcher** in the header to demonstrate instant UI localization. Switch back to English.
4. **Upload a clear selfie** (drag-and-drop or click to browse — JPG, PNG, HEIC, WebP all accepted). Click **"Analyse My Skin"**.
5. *(If returning user)* Point out the **"Your Progress"** sparkline chart showing prior sessions — with ↑ Improving / ↓ Declining / → Stable indicator. Click **"View Details"** to show the History Modal.

> **Tech Callout:** The frontend auto-upscales any image below 1080px short side using Canvas before sending to Perfect Corp, ensuring the HD Skin Analysis API always receives a qualifying image.

---

## ⏱️ 0:45 - The Tech (Analysis Screen)
**Talking Track:** *"Behind the scenes, we're hitting the Perfect Corp API to analyze 14 distinct skin concerns in real-time. We don't just give a score; we show the user exactly what the AI sees."*

**Actions:**
1. Point out the **two-stage Loading States** — first "Analysing your skin…" (Perfect Corp API handshake), then "Curating your personalized routine…" (Gemini + Open Beauty Facts).
2. When the Analysis Screen loads, show the **Overall Health Score** and **Estimated Skin Age**.
3. **Crucial Step:** Click on one of the "Needs Attention" metrics (e.g., *Wrinkles* or *Eye Bags*).
4. Show the **AI Concern Map Overlay Modal** and use the slider to blend the heatmap with the original photo.

---

## ⏱️ 1:30 - The "Wow" Factor (Simulation & VTO)
**Talking Track:** *"Knowing your score is good, but seeing the potential is better. We use Perfect Corp's Simulation API to generate a clinically healed version of the user's skin. And because we want them to feel their best, we've integrated the Makeup VTO API."*

**Actions:**
1. Click **"Generate Care Plan & Simulation"**.
2. On the Simulation Screen, drag the **Before/After Slider** back and forth to show the healed skin. (Works on mobile touch too.)
3. Click the **"💄 Try Signature Red"** button. Explain that this hits the Perfect Corp Makeup VTO API to seamlessly overlay lipstick on the healed result — the slider now compares your original face with the made-up simulation.

---

## ⏱️ 2:00 - The Business Model (Real Commerce)
**Talking Track:** *"How do we monetize? We take the raw API scores and feed them into Gemini 2.0 to generate a personalized skincare routine. Then, we query the Open Beauty Facts API to find real, purchasable products."*

**Actions:**
1. Scroll down to the **Product Recommendations**.
2. Show the severity emoji (🔴 / 🟡 / 🟢) on each product card and the **Key Ingredient** (e.g., Salicylic Acid) Gemini identified based on their specific acne score.
3. Show the actual product cards fetched dynamically from the **Open Beauty Facts API** (complete with brand names, images, and **Buy** links).
4. Point out the **AM/PM Routine** generated by Gemini — personalized to this user's skin type and top concerns.

---

## ⏱️ 2:20 - The AI Consultant (Chat Assistant) ✨ NEW
**Talking Track:** *"Here's what separates SkinIQ from a score dashboard — a contextual AI dermatologist. It knows this user's scores, their routine, and their skin age. Ask it anything."*

**Actions:**
1. Scroll to the **SkinIQ Consultant** chat panel (🤖 icon, below the routine card).
2. Ask a live question: *"Can I use Vitamin C and Niacinamide together?"* or *"What's a budget-friendly alternative to retinol?"*
3. Show the **AI response rendered in rich Markdown** — bullet lists, bold text, comparison tables — not just plain text.
4. Emphasize: the assistant has **full skin context** (overall score, estimated skin age, top concerns, AM/PM routine) injected into every call via Gemini 2.0.

---

## ⏱️ 2:40 - Retention & Export
**Talking Track:** *"To keep users coming back, we built localized reporting and local-first progress tracking."*

**Actions:**
1. Switch the language to **French** using the header dropdown. Show how Gemini AI dynamically re-translates the routine and product hints on the fly (the `isTranslating` loader appears briefly).
2. Click **"📤 Share Results"** — uses the native Web Share API on mobile; falls back to clipboard copy on desktop.
3. Click **"🖼️ Save Image"** to generate a branded PNG report card client-side (html2canvas).
4. Click **"📄 Save PDF"** to demonstrate the print-to-PDF report generation.
5. Click **"↩ Start Over"**. Back on the Upload Screen, show the **Progress sparkline** that now has a new data point — history persisted in `localStorage`, no server needed.

---

## 💡 Quick Tips for Judges Q&A

| Topic | Key Points |
|---|---|
| **Privacy** | All history uses `localStorage`. No images are saved to a database — GDPR & HIPAA ready. |
| **APIs Mastered** | 5 APIs orchestrated: Perfect Corp Analysis, Simulation, Makeup VTO, Google Gemini 2.0 (chat + recommendations), Open Beauty Facts. |
| **AI Chat** | Context-aware follow-ups powered by Gemini 2.0 — skin scores, estimated age, and routine all injected into the system prompt. |
| **Localization** | Full i18n across 5+ languages (EN, FR, NL, HI, …). Gemini dynamically re-translates recommendations when the user switches language. |
| **Auto-Upscaling** | Client-side canvas pipeline ensures Perfect Corp HD API always gets a ≥1080px image, regardless of what the user uploads. |
| **Export** | Three export modes: Web Share API, client-side PNG (html2canvas), and browser print-to-PDF (jsPDF). |
| **E-Commerce** | Open Beauty Facts integration proves this is a monetizable funnel — real products with buy links, not mocked data. |
| **No Dependencies** | The in-app Markdown renderer is hand-rolled — zero external parsing libraries needed. |
